Love2D Input绑定实现

2020-03-10

Input.lua 实现

功能

  1. 函数绑定到按键
  2. 设置按键别名
  3. 检测持续按下, 包括 从按下到开始检测的delay 和 经过interval再确认pressed是否正确

实现

数据结构

保存一个键位表 Input.all_keys, 和 love 的键位表完全一致

binds 以用户设置的按键别名为键, 按键集合为值
保存针对不同按键而设立的表

  1. functions 按键按下就会被唤醒
  2. state 当前按键是否按下
  3. prev_state 上次更新的时候按键是否按下
  4. repeat_state 检测持续按下的时候,
    1. 上次按键按下并响应的时间点pressed_time
    2. delay的长度
    3. 当前的状态, 是否还处于delay检测, delay_stage
    4. 按键响应的间隔interval

如此一来, 检查按键是否是当前更新突然按下的, 就是使用 self.state[key] and not self.prev_state[key]

按键绑定

接受key和action参数
可以选择绑定到函数, 或者是字符串
检查action参数类型, 如果是函数, 那么将functions表的键key设置成这个参数
如果是字符串, 将字符串添加到binds对应的键的表中, 如果binds[key]不存在, 则先新建一个

取消绑定key, 以key为参数, 也是一种合理的方法啦,
如果要取消绑定action, 直接self.binds[action] = nil
按道理来说, 用户是有需要, 看到 action 绑定了哪些 key, 并且可以删除他们的, 所以self.binds是一个可以暴露的数据结构

检查按键是否按下

简单方法pressed
pressed方法中有两个功能

  1. 不传入action参数的情况下, 所有按下的按键，如果绑定了函数，就会被调用
    这个过程在update已经被设置了, 用户无需调用
  2. 检测action是否被按下, 就和之前说的, 通过self.state[key] and not self.prev_state[key]

多功能方法down
接收三个参数, Input:down(action, interval, delay)
repeat_state中,

回调hook

新建自己Input的回调函数, 并且 hook 到 love 的回调函数中, love['keypressed']等同于love.keypressed
这个过程在 new 的时候就被设置,

callback 按键检测

<device>pressed 和 <device>released 函数
pressed 被调用, 则设置该按键的state表值为true
released 被调用, 则设置该按键的state表值为false, 以及取消重复检测的表repeat_state

callback - update

调用self:pressed(),
会更新repeat_state,
如果在delay_stage, 检查是否超出delay时长并取消delay_stage
检查距离上次按键响应的时间是否超出interval

最后还有joystick的绑定这里没讲, 回去拿了手柄再来测试一下吧

从类(表)中实例化

因为__index的存在, 可以将一个表的原型设置为当前的表, 这个过程只不过是setmetatable(o, {__index = Class}, 之后返回这个新表

但是这样有一点点缺陷, 每次都会创建一个这样的表, 有点麻烦, 我们可以单纯的使用Base.__index = Base, 然后将Base本身作为元表就好了

当然, 我们可以把这个函数改为冒号形式, 函数体中的Base改为关键字self, 这样即便本身并不是 Base 的表, 通过元表获取并调用这个方法的时候, 还是将自身作为 self 传递到new中, 返回直接继承自身而不是Base的表, 也就是, 和基类无关

其实更方便地, 为了让后面的可自定义初始化, 我们可以把 设置元表 和 初始化 的方法分开
就成了下面这样, 继承Base的类 还可以只自定义 init, 而不用更改已经既定的 new 过程

静态变量：在类表中一开始就声明定义, 并且在new方法中, 不进行复制和更改罢了, 让新建的对象保持对这个变量的持续引用, 称作公共变量更加合适吧

继承

如上的代码, 当一个table Derive使用了Base:new(), 这样他可以作为一个新的原型, 当又一个table D2, 使用Derive:new(), 因为self的缘故, 这里的self就是Derive, 这样一来, 为Derive添加可覆盖的方法, 可以做到继承

多重继承

简单的, 就是把__index(self, k)改为访问一个基类列表的所有基类, 这个的k成员是否存在的函数, 如果存在, 则返回相应的值

接口 Interface Mixins

其实就是多继承了一个只有函数的表, 如果这个Interface 必须要求实现的话, 可以在默认Interface的函数实现中, 抛出一个 error('not implemented')
当然实现方法可以是遍历 Interface 中的函数, 然后把他赋值到自己的表中

私有化

在new函数里面创建函数和状态, 返回一个表, 让要公开的函数映射到表中, 不放在表中的内容则为私有

对偶表示

从表中使用键获取值, 改为 从一个数值表中以表作为键来获取对应的值
之后也在这个数据表中获取值, 让这个数据表变为一个模块的local变量, 也就是私有化, 能做到一定的封装
但是存在一个问题, 即便是对象表被销毁了, 数值表中对应对象表的值并没有销毁, 这时候需要垃圾收集

实现方式

classic.lua

  • Extend - 约定, 以 __ 开头的函数, 为可覆盖/重用的虚函数, 直接复制到新的对象cls, 其他内容使用metatable, 设置超集, 并且在 cls 写下记录 cls.super = self
  • new - 用户自定义, 只是单纯的初始化, 叫做 init 更合适
  • __call - 这个就是用户调用的新建对象的方法了, 直接使用 newclass = Class(), 来新建一个对象, 其实是新建一个元表是 Class 的空表, 然后对这个空表, 调用 new, 来初始化, 最后返回, 这个是适当的做法
  • is - 递归 getmetatable, 不断检测和特定 class 是否一致
  • Implement - 接受可变参数指示多个Interface表, 对每个Interface表, 将其所有函数复制到自身, 这个。。不检测函数重名啊

hump.class

因为 table 和 function 在lua中, 单纯使用赋值运算符, 得到的是引用/地址, 因此如果要deep copy, 需要遍历并且递归表中的元素来进行拷贝

  • include_helper(to, from, seen) 这个递归代码… 其实就是将from表中所有的值都逐个递归拷贝到to中, 如果from中的键包含表, 那么将这个表进行拷贝, 如果from的值是个表, 建立一个空的表作为递归的to参数, 对其进行拷贝并返回
  • include(class, other) 使用include_helper递归函数, 进行拷贝后返回
  • clone(other) 使用include, 新建一个other的深拷贝
  • new 和classic.lua的__call逻辑一致, 新建一个表, 然后对其调用init方法来初始化,

算法竞赛入门经典

2020-03-05

Ch7 暴力求解法

按字典序排列

不使用 STL 的情况下，按从小到大的顺序遍历当前没有被使用过的元素 S，
把元素放进当前排列数组 A的当前元素位置 cur，递归向下继续搜索，
并且当前顺序下依然遍历中，在递归后，被覆写的 A 的范围，不会被当前遍历所访问到，因此子递归并不影响
如果达到集合终点，把之前所有的数组打印出来

也可以使用 STL 中的函数，next_permutation，它在数组已经有序的状况下，更改末尾处，让元素不断与最开始无序的元素的之前的元素进行倒转返回，生成新的排列，直到集合彻底倒序，返回 false

子集生成

以下生成的子集，都包含了自身

为了保证不重复，按顺序来，只选取比当前集合内所有元素更大的元素插入
每次插入新的元素，都进行一次打印，表示当前的子集

因为子集就是，哪些元素依然存在 的组合，
我们每次分成两种情况-当前元素是否存在，再进行向下递归
使用 位向量 来保存哪些元素依然存在，生成子集，

更加简单的，就是使用二进制位，表示当前元素是否存在，
因为子集的数量就是 $2^n$，我们取从 $1 - 2^n$的数字，列出集合
判断 i 位 是否存在，来相应地表示集合 第 i 个元素 是否存在

回溯法

八皇后谜题

因为八皇后不可能有重复的列/行，因此每行每列，都有且仅有一个皇后
由此我们定义 数组 A[n]，索引表示列数，数组索引的元素 表示 当前列元素所处的行

我们可以生成所有的排列，然后检查所有元素互相是否不在同一对角线上。
但这样每次生成排列数量会多，而且每次检查消耗时间也多

我们可以在生成排列的同时，就检查当前元素和之前所有的元素是否都不在同一对角线上，
如果失败就返回，不继续生成这样的排列，这就是 回溯

如何检查和之前元素是否在同一对角线：

每次都要遍历之前的元素来检查是否重合，实属浪费时间，
我们可以每次放置一个皇后，就把这个皇后所占据的行，对角线，副对角线都进行标记
但要切记，每次放置并标记，使用递归之后，要将之前的标记重置再继续

应用 - 素数环

一个小细节是，我们要注意素数环从 1 开始，而数组下标从 0 开始，
最好给自己下个规定，数组就从 0 开始，元素取当前索引 + 1
