Unity 脚本的序列化

script-Serialization-BuiltInUse.html

Unity 使用序列化来从硬盘中加载和存储 场景，Assets，AssetBundle

当我们修改了 GameObject 的数据的时候，在 Inspector 窗口会重新序列化，来显示出数据

脚本序列化 热重载

在 Unity 处于启动状态的时候（无论 Editor 还是 runtime），当脚本发生改变或者添加的时候，Unity 会先保存一份序列化数据，然后重新加载脚本，之后将之前存储的数据反序列化放回脚本中。官方称之为 hot reloading

在 Inspector 中查看和修改的数据，并不会调用任何的 属性 getter / setter

序列化规则

首先必须符合以下规则

  • Is public, or has a SerializeField attribute
  • 非 static
  • 非 const
  • 非 readonly
  • 具有符合如下规则的字段

能够变序列化的简单类型

  • 带有 Serializable attribute 的自定义非抽象、非泛型 class 类型，并且避免内部的循环引用（见下方）
  • 带有 Serializable attribute 的自定义 struct 类型
  • 指向 继承 UnityEngine.Object 的类型 的引用
  • 原始数据类型 (int, float, double, bool, string, etc.)
  • Enum 类型
  • Unity 内置类型: Vector2, Vector3, Vector4, Rect, Quaternion, Matrix4x4, Color, Color32, LayerMask, AnimationCurve, Gradient, RectOffset, GUIStyle

能够被序列化的容器类：
存储 Simple field types 的数组 或者 List

序列化器可能会导致错误的结果

让序列化的多个class 指向同一个引用

不支持序列化引用为 null 的自定义 class

当 Unity 碰到 null 的对象的时候，会给这个对象新分配一个空间
如果这个对象内部还有自定义 class，则继续往深处 new 新的对象，这个次数不会超过 7次，否则停止序列化，这是为了防止循环引用的问题，

不支持多态

比如一个 public Animal[] animals 字段，但是对其放入了 Animal 的继承类，Dog Cat 之类的，那么它们都会被序列化成 Animal

建议

优化地使用序列化

  • 以让 Unity 序列化最少的数据 为目标，来组织你的数据布局
  • 永远不要让 Unity 序列化重复的数据
  • 避免带有嵌套，循环引用的 class 类型
