故事编写

注释语法 - 在一行的开头添加 // 即可

// 这是一行注释

之后的代码块中，使用<>包裹的内容，就是编写者可自定义的内容，实际编写要去掉 <>

节点和内容

所有的对话存储到后缀名为 .yarn 的文件下，可以使用 文本编辑器，或者官方的 在线编辑器 来进行创建和编辑

节点（Nodes）

一个 yarn 文件内部由 节点 组成，每个节点至少包含一个 标题 和一个 正文，
标题很重要，就是该节点的名称，被用于选项的跳转
更具体的，由 header 头部 和 content 正文内容 组成，标题是 头部 的一部分，

标题必须是英文且不能有空格，必须在整个程序中具有唯一的名称，这个有待商讨怎么进行节点标题的命名（角色的拼音之类的？）

语法

头部是一系列 冒号分隔的一对内容，比如说title: 标题名称，tags: 标签1 标签2
在新行使用 --- 将头部和正文分开，在节点末尾使用 === 表示这个节点的结束

示例

yaml

节点正文内容

节点内容由三种类型的 文本行 组成 - 对话行，命令行，选项行

对话行

语法如下，冒号注意采用半角（英文输入法中的），一句话不要换行，之后会自定义格式让对话内容的格式更加丰富

<人物名称>: <对话内容>

在 YarnSpinner 运行的时候，会把对话行一行一行交给游戏

当然也可以省去人物名称，以对白的形式呈现

命令行

以<<和>>包裹的一个命令，
命令以一个单词开头，如果这个命令需要参数，在命令单词后，填入空格后添加，可以添加多个参数
比如下面的 close 命令，和 wait 命令、1 作为参数，表示等待 1 秒钟

yaml

所有的命令词需要程序上的支持，以调用游戏内部的交互，策划可以提出需求，需要怎样的演出效果

选项行

当你想让角色 做选择、开启分支对话 的时候，可以使用 选项行

语法

[[<玩家所看到的选项文本>|<要跳转到的节点的名称>]]

示例

yaml

控制对话流

选项

在一个节点的正文中，yarn spinner 会收集所有的选项行，在到达节点末尾的时候，将所有的选项显示出来，给玩家选择
因此选项行通常写在节点的末尾，虽然写在任何地方都可以，选项是按上下顺序显示在玩家面前的

当节点结束的时候，没有任何选项被看见，yarn spinner 会停止对话

快捷选项

上面的 选项 功能，必须在节点末尾进行收集和响应，限制挺多的
当如果想要只在一个节点中，写出短小的分支内容和选项的时候，那么就可以使用 快捷选项（Shortcut Options）

语法则是采用一个 ->（连接符/减号 和 大于号），空格后接选项文本

选项子内容

快捷选项，可以有属于这个选项自己的内容行，当这个选项被选中的时候，则会被一行一行执行
这样的 子内容行，需要在 快捷选项的下一行进行换行然后再开始编写内容

yaml

嵌套 快捷选项

快捷选项是可以嵌套的，只要之后的内容，相比

建立复杂的嵌套对话结构很简单，但是这可能导致你的文件内容太多太复杂 难以阅读
当出现这种情况的时候，建议采用常规 选项

条件语句

你可以让 快捷选项 在一定条件下，才会给玩家展现和选择
只需要在快捷选项行的末尾，加上 if 命令语句

语法和选项很相似，但是不是在末尾的时候才展现出来，而是在一遇到就执行

表达式 和 if 语句

if 语句用来控制 一部分内容 是否显示
if 语句的使用方式，是以 if 为命令词，后接一个 表达式，当这个 表达式 的结果为 真 的时候，那么进入 if 命令和 endif 命令包含起来的块

$gregg_day_intro is 0 是 if 语句中用到的表达式，$gregg_day_intro 是一个 变量，is 是一个运算符，可以理解成数学中的 =，判断是否相等 / 左侧内容是否 是/is 右侧内容

更完整的逻辑判断，当 if 得到的结果 为 假 的时候，会往下查找是否存在 elseif 或者 else 语句
elseif 当上一个 if / elseif 判断失败后，进行这一行的判断
else 语句，在之前所有的 ifelseif 得到的结果都为假的时候，才会进入它所属的块
规整的顺序，是将 else 写在最后一个判断，elseif 写在 if 判断之后
