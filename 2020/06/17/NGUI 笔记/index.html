NGUI 杂项笔记
Created 2020-06-17| Updated 2020-06-17
Post View:

任意组件，父节点必须有 panel 才可以在游戏中被看到

设置 Sprite

使用 UI2DSprite 可以直接设置原始的 Unity.Sprite 类型

使用 UISprite 需要接收一个 Atlas
Atlas 需要使用 Atlas maker 窗口来创建 atlas，并为 atlas 增减子图像
Atlas 以减少 draw call 为目的，把一系列图片整合为一张图片

布局系统

想要让 UI 变得 无视 DPI（分辨率） 和拉升比例，可以设置 UI Root 中的 scaling style
比如 Constrait，

这些布局都只能排布 基于 UIWidget 的类
同时，因为排布的内容是通过 Transform.GetChild 获取的，包含了自身，所以，带有 Table / Grid 插件的对象，不要包含 Widget！

Grid

Smooth Tween 让其更新布局的时候会非常平滑地移动控件

Grid 和 Table 的区别

At the top of each of those classes we have a summary of what it does.

UIGrid “All children added to the game object with this script will be repositioned to be on a grid of specified dimensions.”
UITable “All children added to the game object with this script will be arranged into a table with rows and columns automatically adjusting their size to fit their content”

So UIGrid will just arrange the items into formation. UITable will resize the elements.

grid 只是整理控件，而 table 会拉伸它们以保证对齐

交互系统

UICamera 管理一切的事件，
他会检查所有具有 Trigger Collider 上面的脚本，是否具有 OnClick 等等 函数，在相应事件下调用他们
这其中调用了 SendMessage

UIButton

一个 Button 需要一个 BoxCollider 用来接收事件和信息
或者使用代码的方式 NGUITools.AddWidgetCollider(optionBtn.gameObject);，来直接添加自适应按钮
上层的 BoxCollider，会覆盖下层的 BoxCollider，这种问题出现在，如果一个 Label 位于一个按钮之上还设置了 BoxCollider 的话，那么当鼠标移动到 Label 的碰撞区域内，下面的按钮的碰撞区域将不会得到响应

如果想要 Button 响应到 同时这个 UISprite 还有另一个物体的话，那么我们可以再新建一个 UIButton，然后再设置它的 TweenTarget 为 Button

使用 UIButton.current 可以得到当前活跃的 Button

OnClick 可以同样应对带有参数的函数，这样的参数能在 Inspector 中赋值，甚至可以应对 gameobject 类型里面嵌套的类型，比如 一个函数接收 string，我们可以选择 给其赋予 GameObject( ToString() ), gameobject.tag, transformt.tag 等等

Tween

即使另一个 Tween 并没有被 Enable，其中的方法被 Notify 了以后，依然会自动 Enable

使用 Play Tween 来取代 Button 唤醒其他的 tween，内部有很多选项

UIEvent Trigger

这个脚本包括了很多事件响应，比 Button 的元素更简单，

UIKey Binding

将 交互控件 的响应操作，绑定到按键上，比如 UIButton 的 click

UI Input

On Submit，表示了用户按下 Enter，把 Input Box 中的文本上传上去时候，要执行的其他动作

UIKey Navigation

还挺麻烦的。。。用于响应上下键来进行锁定操作

Drag And Drop

UIDragDropItem
使得这么一个物体可以被拖拽移动，并且可以设置 Restriction，让鼠标在什么 情况 / 方向上 拖拽才能生效

UIDragDropRoot
拖拉中的物体，会临时成为具有 该 component 的物体的 child，释放控件后 transform.parent 又会恢复

UIDragDropContainer
可以让拖拉中的物体放置到具有该脚本的空间上（需要 BoxCollider），当然也可以 Redirect，指向其他的母控件

UI 设计实现

Tab 实际上是一个 UIToggle，点击的时候开启，不点击的时候关闭

数据绑定 - Property Binding

因为这个绑定对于 Source 和 Target，必须类型可转换，还不可以添加自制的 类型转换器

因此要自定义进行转换的话，使用 PropertyReference 来从 Inspector 中设置对象属性
然后使用 Get() Set() 方法，来获取一方的属性，然后设置给另一方，在这中间，可以自定义自己的 中转方法

封装了的代理方法 EventDelegate

NGUI 控件上的 响应方法，比如 onClick，通常是一个 List
要为其添加通常的 Action 代理，只需要使用静态方法 EventDelegate.Add(, ) 即可

UIButton 会不知不觉的更新 Sprite，普通情况下，建议使用 UIButton Color

UIGrid 布局，需要手动指示它进行重新排布，使用 Reposition() 方法
